\# Библия Проекта: Viral Engine by Sherlock

\## Документ 03: User Journey \& UI/UX (Путь Клиента и Интерфейс)



\### 1. Концепция "The Master Switch" (Главный переключатель)

Интерфейс строится вокруг одной центральной консоли управления. Клиент не бегает по вкладкам "Видео" или "Тексты". Он работает с \*\*Ядром Смысла\*\*, из которого распускаются ветви разных форматов.



\### 2. Подробный User Journey (Шаг за шагом)



\#### Шаг 1: Инициация (Input)

\* \*\*Действие:\*\* Клиент вводит тему, вставляет ссылку на видео/статью или выбирает идею из блока "AI Suggestions".

\* \*\*Экран:\*\* Чистый Dashboard с одной строкой ввода и списком карточек-рекомендаций.



\#### Шаг 2: Работа со Сценарием (The Script Lab) — КРИТИЧЕСКИЙ ЭТАП

\* \*\*Действие:\*\* ИИ выдает "Мастер-сценарий". Это текстовая структура, разделенная на логические блоки (Хук, Тело 1, Тело 2, Вывод, CTA).

\* \*\*Инструментарий:\*\* - Прямое редактирование текста.

&#x20;   - Кнопки быстрой модификации: "Сделай короче", "Добавь юмора", "Больше экспертности".

&#x20;   - Чат-окно для правок: \*"Добавь сюда упоминание моего курса по автоподбору"\*.

\* \*\*Результат:\*\* Клиент нажимает кнопку \*\*"Утвердить сценарий"\*\*. Без этого шага система не списывает кредиты за продакшн.



\#### Шаг 3: Визуальная Раскадровка (Storyboard)

\* \*\*Логика:\*\* Система берет утвержденный текст и генерирует набор статичных изображений через Flux 1.1 (сверхдешево и фотореалистично).

\* \*\*Интерфейс:\*\* Клиент видит горизонтальную ленту карточек.

&#x20;   - \*Карточка 1:\* Фото клиента (Аватар).

&#x20;   - \*Карточки 2-5:\* Сгенерированные фото-иллюстрации для B-roll.

\* \*\*Действие:\*\* Если фото не нравится, кнопка "Regenerate" меняет картинку за 0.01 кредита. Клиент понимает: "То, что я вижу сейчас — это то, что оживет в видео".



\#### Шаг 4: Запись или Генерация A-Roll (Studio Recording)
Система предлагает два пути создания "говорящей головы":

1. \*\*A-Roll Studio (Teleprompter):\*\* 
   - Клиент сам записывает себя, используя встроенный телесуфлер. 
   - \*\*Инновация:\*\* Параллельная запись аудио-дорожки для мгновенной транскрипции и экономии RAM на мобильных устройствах.
   - \*\*Review:\*\* Возможность мгновенного переснятия дублей ("Retake Masterpiece").

2. \*\*AI Avatar (Faceless/HeyGen):\*\* 
   - Автоматическая генерация через HeyGen (Talking Head) для тех, кто не хочет быть в кадре.

\#### Шаг 5: Мульти-генерация и Монтаж (Omni-Production)
После утверждения записи или выбора аватара происходит параллельный процесс:

1. \*\*Assembly (Монтаж):\*\* ИИ автоматически расставляет B-Roll (футажи), накладывает субтитры и синхронизирует аудио.
2. \*\*Text:\*\* На базе сценария пишутся: Пост для TG, Тред для X, Статья для LinkedIn.
3. \*\*Gallery:\*\* Фото из сториборда оформляются в стильную карусель для Instagram.



\#### Шаг 5: Выдача (The Delivery)

\* \*\*Telegram:\*\* Пользователь получает уведомление. В боте появляются 3 сообщения:

&#x20;   1. Видеофайл (MP4).

&#x20;   2. Текстовые посты (Copy/Paste ready).

&#x20;   3. Архив или альбом с фото для карусели.



\### 3. Элементы UI (Спецификация для Antigravity IDE)



\* \*\*Dashboard:\*\* Темная тема (Deep Space Blue/Black), акцентные цвета — "Sherlock Gold" или "Neon Mint".

\* \*\*Credit Widget:\*\* В углу всегда виден баланс: "У тебя 840 кредитов (\~8 полных паков контента)".

\* \*\*Status Stepper:\*\* Прогресс-бар сверху: `Идея -> Сценарий -> Раскадровка -> Рендер -> Готово`.

\* \*\*The "Magic" Button:\*\* Кнопка финализации должна быть массивной, с эффектом свечения, дающая ощущение запуска мощного двигателя.



\### 4. Режим "Pro" (BYOK)

Для крупных клиентов в настройках есть тумблер: "Использовать мои API ключи". При включении интерфейс не меняется, но списание внутренних кредитов за рендер прекращается, берется только фиксированная абонентская плата за "Интеллект системы".
