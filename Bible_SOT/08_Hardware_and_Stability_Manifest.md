# Библия Проекта: Viral Engine by Sherlock

## Документ 08: Манифест Стабильности Оборудования и OOM Protection

Данный документ фиксирует технические стандарты работы с аппаратной частью (камера, микрофон) и методы обеспечения стабильности приложения на мобильных устройствах.

### 1. Стратегия захвата медиа (Recording Suite)

#### 1.1. Dual-Recording (Двойная запись)
Для обхода тяжелых операций извлечения аудио из видео на смартфонах, приложение записывает два параллельных потока:
- **Video Track**: Записывается в `video/webm` (Android) или `video/mp4` (iOS).
- **Audio Track**: Параллельно записывается в легкий `audio/webm`.
- **IndexedDB**: Оба блоба мгновенно сохраняются в локальное хранилище `MediaBuffer` под ключами `pending_upload_${projectId}` и `pending_audio_${projectId}`.

#### 1.2. Управление ресурсами (Forced Release)
Для предотвращения утечек памяти (OOM):
- При переходе в режим "Монтаж" (Assembly) вызывается функция `stopCamera()`, которая принудительно останавливает все `MediaStreamTracks`.
- Удаляются глобальные ссылки на рекордеры (`window._audioRecorder`).

### 2. Защита от зависаний (Mobile Resilience)

#### 2.1. Infinity Duration Guard
На Android браузер Chrome часто возвращает `Infinity` для длительности WebM-файлов.
- **Стандарт**: Любые циклы отрисовки шкалы времени (Timeline) обязаны проверять `isFinite(duration)`.
- **Fallback**: Если длительность не определена, использовать 60 секунд как безопасный дефолт.

#### 2.2. Async Chunked Audio Processing
Прямая обработка аудио-данных (конвертация в WAV) блокирует основной поток JS на 30+ секунд.
- **Решение**: Использование "чанковой" обработки (Chunked Processing) — цикл обрабатывает данные порциями (по 50к сэмплов) с микро-паузами `setTimeout(0)`. Это позволяет интерфейсу оставаться отзывчивым.

#### 2.3. Stability Heartbeat
В режиме обработки отображается визуальный индикатор (пульсирующая зеленая точка), обновляемая через `requestAnimationFrame`. Это позволяет пользователю и разработчику мгновенно понять, "жива" ли страница или поток JS заблокирован.

### 3. Настройки кодеков (Compatibility Matrix)

- **Android**: Приоритет `video/webm;codecs=vp9,opus`.
- **Bitrate**: 2.5 Mbps для мобильных устройств, 5.0 Mbps для десктопа.
- **Audio Resampling**: Нативное понижение частоты до 16kHz через `OfflineAudioContext` для совместимости с Whisper API.
